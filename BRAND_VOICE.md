# Brand Voice

How ShowUp writes — and what it never does.

ShowUp has **two distinct copy registers** that coexist but never mix. Knowing which one you're writing in is the first decision before any word is chosen.

---

## The two registers

### 1. UI register — factual, neutral, specific

Used in: buttons, labels, status messages, metadata, error states, form fields, settings, notifications.

This is the calm voice of the product. It states what is, never what should feel. It does not pretend to be your friend. It does not perform enthusiasm. It is a coordination tool speaking like a coordination tool.

**Rules:**

- No emoji. Ever.
- No exclamation marks.
- No "we", "us", or "our" — ShowUp is not a community brand.
- No filler words: "let's", "looks like", "almost", "just", "simply".
- Specific over generic: `Sunday · 9:00 AM · West Vancouver`, not `This weekend in your area`.
- Quantitative when possible: `2 more needed`, not `Almost there`.
- Imperative for actions: `Create event`, not `Let's create an event`.

**Examples (use → not):**

| Use ✓ | Avoid ✗ |
|---|---|
| Quorum reached | Confirmed! 🎉 |
| 2 more needed | Almost there! |
| Sunday · 9:00 AM · West Vancouver | This weekend in your area |
| No events yet | Looks empty around here! |
| Create event | Let's create an event |
| View my events | See all → |
| Event cancelled · did not reach quorum | Aw, this one didn't make it 😔 |
| You're confirmed | You're in! |
| Saved | All set! |

### 2. Marketing register — conceptual, verbal system

Used in: landing pages, advertisements, onboarding screens, social media, app store listing, promotional emails, the tagline.

This is the conceptual voice of the brand. Where the UI register is calm and factual, the marketing register is playful with language — but always disciplined, always anchored in the same construction.

**The verbal system: "Show ___ ... ShowUp"**

The brand name "ShowUp" carries a double meaning in English:
- **show up** (verb) — to appear, to physically arrive
- **show** (verb) — to demonstrate, to reveal

Marketing copy uses this by setting up the construction `Show ___` (with a variable word filling the blank) and resolving it with `ShowUp` as the punchline. The pattern works because every line lands on the same word — the brand name doubles as the call to action.

**Pattern examples (read each row as a sequential animation, not a static lockup):**

> `Show|` → `ShowCasually` → (deleted) → `ShowSporty` → (deleted) → `ShowUp.`

> `Show|` → `ShowHiking` → (deleted) → `ShowCoffee` → (deleted) → `ShowUp.`

> `Show|` → `ShowQuiet` → (deleted) → `ShowEarly` → (deleted) → `ShowUp.`

**The animation: typing as the conceptual mechanism**

The marketing system is not a static lockup — it is an **animation**. The verb construction "Show ___" plays out as a typing sequence that visualises hesitation, deletion, and arrival. This mirrors the actual mental state of a user deciding whether to attend an event: considering, second-guessing, considering again, and finally — confirming.

The animation sequence:

1. The word `Show` appears on screen with a blinking cursor `|` after it, as if someone is about to finish typing.
2. A variable word is **typed out letter by letter** after `Show` — for example, `ShowCasually`. The blinking cursor stays at the end.
3. The variable word is then **deleted letter by letter**, as if the writer changed their mind. Back to `Show|`.
4. A second variable word is typed (e.g. `ShowSporty`), then deleted the same way.
5. Optionally a third (e.g. `ShowQuiet`), then deleted.
6. Finally, `Up` is typed — `ShowUp.` — and the cursor stops blinking. The wordmark settles.
7. **The wordmark rises slightly** with a soft motion, and the tagline appears beneath: `That's all it takes.`

This is the canonical brand animation. Static lockups should be avoided in favor of the animated form whenever the medium supports it (video, web, app onboarding, social motion graphics). When a static fallback is unavoidable (print, app store screenshots), use the wordmark `ShowUp` alone with the tagline beneath.

**The lime accent in marketing animation:**

The accent color (`#D1F471`) appears as the **blinking cursor** during the typing sequence. The cursor IS the lime — that single small mark is the only chromatic element on screen during the entire animation, representing the moment of decision in real time. When the animation resolves on `ShowUp.`, the cursor stops blinking and disappears, the lime momentarily holding the period, then fading. The tagline that follows is in plain ink.

This is why the lime works conceptually: it is **literal hesitation** — a blinking input cursor — rather than a decorative mark. It earns its place by representing the very thing the product is about: the pause before someone commits.

**The tagline:**

> **That's all it takes.**

Used beneath the resolved wordmark as the closing line of every campaign animation. The full statement reads `Show up. That's all it takes.` but in the animation, "Show up" is performed by the typing sequence itself, so the tagline only needs to deliver the second half.

**Rules for marketing register:**

- The construction always resolves on `ShowUp` — never break the pattern by ending on something else.
- Variable words should be **single words**, not phrases. `ShowHiking` works; `ShowGoingHikingThisSunday` does not.
- Variable words should be **active or descriptive states**, not nouns of objects. `ShowCasual` (a state of being) works; `ShowCoffee` (an object) is borderline acceptable when the object stands in for an activity, but `ShowChair` would not.
- Marketing copy is always sentence case, never Title Case or ALL CAPS.
- Even in marketing, no emoji.

## Strict separation: never mix the registers

The UI never uses the marketing voice. The marketing copy never lives inside a button.

**Wrong:** A confirmation button that says `ShowUp to confirm` (marketing voice in UI surface).
**Right:** The button says `Confirm`. The marketing voice lives on the landing page that brought the user to the app.

**Wrong:** A landing page hero that says `View available events near you` (UI voice on marketing surface).
**Right:** The landing page hero shows the typing animation that resolves on `ShowUp.` followed by the tagline.

This separation matters because the two voices serve different purposes. The UI voice reduces friction; the marketing voice creates memorability. Mixing them dilutes both.

## Tone by surface — quick reference

| Surface | Register | Example |
|---|---|---|
| Button | UI | `Create event` |
| Status badge | UI | `Quorum reached` |
| Empty state | UI | `No events yet` |
| Error message | UI | `Couldn't load events. Try again.` |
| Push notification | UI | `Quorum reached for Sunday hike` |
| Onboarding screen title | Marketing | `ShowUp.` (or animated typing sequence) |
| Landing page hero | Marketing | Typing animation: `Show` + cursor → `ShowCasually` → deleted → `ShowSporty` → deleted → `ShowUp.` |
| App store description | Marketing (opening), UI (feature list) | Mix both registers, marketing first |
| Tagline (any surface) | Marketing | `That's all it takes.` |
| Email subject (event reminder) | UI | `Sunday hike — 9:00 AM tomorrow` |
| Email subject (campaign) | Marketing | `ShowUp.` (paired with animation in body) |
| Static fallback (print, app store screenshots) | Marketing | Wordmark `ShowUp` + tagline `That's all it takes.` — no dot, no lockup |

## The forbidden words list

Words and phrases that never appear in ShowUp copy, in either register, because they break the brand's neutral positioning:

- **"Community"** — ShowUp is not a community. It's a coordination tool. People who attend an event are not part of a "ShowUp community."
- **"Friends"** — the product doesn't promise friendship. People who show up to the same event are co-attendees, not friends.
- **"Network" / "networking"** — ShowUp is not LinkedIn. The product has no professional framing.
- **"Match" / "matching"** — ShowUp is not a dating app. People are not matched; they confirm attendance to the same event.
- **"Discover your tribe"** — too tribal, too clichéd, and false (see "community" above).
- **"Connect with like-minded people"** — generic, and again implies social bonding the product doesn't promise.
- **"Vibes"** — too casual; clashes with the editorial tone.
- **"Crush it" / "level up" / "unlock"** — gamification language doesn't fit. ShowUp is calm, not a self-improvement app.
- **"Welcome to the family"** — see "community."

## Do's and don'ts cheatsheet

✅ **Do:**
- Lead with the specific time, place, activity.
- Use numbers when they mean something (`6 / 4`, not `Lots of people`).
- End sentences with periods, not exclamation marks.
- Trust the user to read short copy — don't pad.

❌ **Don't:**
- Speak as a brand persona ("ShowUp loves...", "We think...").
- Add reassurance copy users didn't ask for ("Don't worry, your data is safe!").
- Fake intimacy ("Hey there, friend!").
- Use cute synonyms for normal actions ("Roll up", "Hop in", "Jump on").
- Say what users are feeling ("Excited about this one?").
